Intern-菜鸟供应链网络

更新于: 2019-04-07 | 分类于 intern

周二投递了菜鸟网络区块链组的实习,今天收到了面试官的电话面试,下面总结一下这次面试.

首先是面试流程:

  • 自我介绍
  • 介绍学校的项目(区块链)
  • 区块链坊方面的问题
  • 分布式系统方面的问题
  • 网络方面
  • 数据库方面的问题
  • 操作系统方面
  • DSA 方面的问题
  • C++ 方面的问题
  • Python 方面的问题
  • 公司与工作介绍

面试情况

自我介绍

和上次相比,我介绍的内容没有很大变化:学校-年级-专业-研究方向-项目经历.

复盘来看需要改进的地方有: 自我介绍部分还是需要准备一点材料 语言表达的不够精炼 思维的逻辑性可以加强 遣词造句需要推敲一下

介绍学校的项目

这一点比上次做得好,主要介绍了 TOKEN 和 bitcoin 的研究,突出了研究的成果,同时补充了正在进行了 Fabric 的研究,对 Fabric 的认证机制和共识机制做了分析,面试官在着一块没有进行深入的考察

这里可以再好好梳理一下,整理一个思路清晰的材料

区块链方面的问题

这里分为两个领域:

  • Fabric
    • MSP 机制:解释一下 MSP 是干什么的
    • kafka:解释一下 kafka 的工作机制
  • bitcoin
    • 一致性:比特币区块链是如何保证一致性的
  • 区块链基础
    • 共识算法:了解哪些共识算法
    • 区块链的存储机制
      • 文件系统, LevelDB, CouchDB
      • PoW,PoS,DPoS,BFT,EBFT,Alagond,…

区块链方面比较熟悉,存在的问题如下:

  • 逻辑不清晰,语言不简练
  • bitcoin 方面的内容还需要好好总结一下
  • 对基础知识要熟练掌握

分布式系统方面:

  • 多线程编程:
    • 悲观锁\乐观锁(看到了,没去研究 :))
    • 不使用锁机制,设计一个多线程访问 cache 的方案?(面试官提示用 cyclebuf)

结合分布式系统好好整理一下基础知识点

网络方面

  • TCP
    • 解释 三次握手,四次挥手
  • 对于 C/S 模型,假设二者已经建立了连接,此时服务器宕机并重新上线,请问客户机还能使用原来的连接吗?
    • 我认为不能,服务器的套接字释放了

  • 使用过 tcpdump 吗

    • 我用 wireshark, Linux 下网络测试的软件有很多

    看来需要加强 TCP/IP 方面的积累

数据库方面

  • 使用过什么数据库软件?
    • 我学的是数据库原理(:)),没有熟练使用 SQL 数据库(我估计他是想问 k/v Model)
  • 数据库的存储原理
    • B+ 树
  • 解释一下 B+ 树
    • 没用过,看了就忘了
  • 还有一个高级机制…等我想一想

数据库方面要回顾一下

操作系统方面

  • 用过 Linux 吗?
    • 我的工作环境就是 Linux
  • 在 Linux 中获取线程的资源情况的命令是?
    • ps + 管道? 面试官说用 strace

Linux 方面需要再回顾一下,把内核那块再好好看一看

DSA 方面

  • 说说有哪些排序算法?
    • 冒泡,插入,归并,快排

  • 冒泡算法的时间和空间复杂度?

    • O(n^2), O(1)

  • 结合笔记回顾一下

  • 在 Leetcode 上多练习一下

C++ 方面

  • 问了一个 C++ 的高级特性,忘了(面试官应该是玩 JAVA 的)

语言问题我想来淡化一下,在基础能力上加强一下

Python 方面

  • Python 用过什么库?
    • 我介绍了两大类:爬虫和机器学习
  • 介绍一下你的爬虫项目,遇到的一些反爬机制
    • 这里需要总结的是,如何介绍自己的项目?并引导面试官的思维?
    • 这里提到了两个问题的处理:验证码,IP池

公司情况

浙江菜鸟供应链金融股份有限公司,base 杭州, 这是唯一一个区块链的组,做供应链金融,面试官介绍区块链做供应链金融算是比较好的切入点了.

工作情况

问我以后工作的打算,有没有希望 base 的地点,当然是四海为家了, 面试官介绍说他们这里的工作 80% 都会是 Java 开发,真的做区块链的还是比较少,我这里当然是没有意见的啦(有可能被套路了).

还有工作时间的问题,希望能够实习的久一点,按我的计划预计 5 月中旬就能结束手上的工作.

后续

我走的内推通道,和今年的大面积实习生招聘是一个流程,后面的情况会用邮件通知.

那么复盘情况是:

  • 准备互联网那一啪啦的东西
  • 加速科研进度,以开发为主
