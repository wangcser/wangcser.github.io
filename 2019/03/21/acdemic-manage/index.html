我的（跨平台、多设备）文献管理方案

更新于: 2019-04-07 | 分类于 Efficency

不得不承认，工具对人类的意义是颠覆性的，它不仅提升了工作效率，还改变了工作方式，甚至看待问题的角度

看，那个人还在钻木取火诶

今天总结一下我的跨平台文献管理方案，为大家的文献管理提供一点思路。

实现这样的方案需要一些准备材料，他们包括：

  • Zotero：一个开源免费文献管理软件（项目），直观的交互和还能忍受的界面是我选择它的理由，他也是整个方案的核心
  • Zotero for Chrome：让你一键完成 “右键-另存PDF到本地—导入PDF到管理软件-构建文献索引” 这种日常操作
  • Google Scholar Extension：让你一键完成 “获取文献关键词-打开 Google Scholar-键入关键词并检索-获取文献 PDF”
  • OneDrive：微软出品的云同步盘，跨平台的核心，当然你可以用其他云同步盘来替代

整个方案分为两个部分：

  1. 使用 Zotero 高效管理文献

  2. 在不同的设备上同步 Zotero 数据库

  3. 使用 Zotero 高效管理文献

首先解决使用问题

使用 Zotero 管理文献很简单，首先打开软件，然后将本地文献文件拖入软件即可，之后 Zotero 会自动基于在线数据库为你的文献创建索引（索引包括但不限于：标题，作者，摘要、期刊…），你也可以为你的文献创建笔记，添加标签，使用文件系统对文献进行分类… 。

然后解决高效问题

如果你准备好了前文介绍的材料（安装并调试好了相关软件），这里我只需要为你描述一下整个操作场景就可以了：

  1. 在知乎看到一篇关于分布系统统领域经典文献的综述回答，我需要获取 GFS、MapReduce、BigTable 的文献
  2. 在文章中左键选取需要的关键词，之后单击浏览器上的谷歌学术快捷按钮，此时在网络状态良好的情况下，你会得到一个相关文献列表
  3. 选择最相关的那篇文献，点击 PDF 按钮跳转到相关页面，如果你运气好的话，你就看到了你需要的文献
  4. 在PDF 文献页面点击 Save to Zotero 按钮，选择你希望存储的目录，等待 Zetero 工作完成即可（请保持本地 Zotero 启动），此时你在 Zotero 中可以看到文献已经加入你的学习大礼包了
  5. ctrl + w 关闭 PDF 页面，结束战斗或者跳转步骤 2 😄

如果你只在一个设备（工作笔记本或者台式机）上进行文献管理，OK，今天的安利结束，祝您科研愉快

如果你有跨平台、多设备的需求，那么可以再往下看看 ⬇️

  1. 我在研究分布式系统的经典工作，因此上知乎看看大佬们对这个领域的看法

  2. 跨平台、多设备地使用 Zotero

首先介绍一下我的需求：

  • Windows 笔记本的管理需求
  • Win / Linux 双系统 PC 的管理需求
  • Pad 的文档阅读需求
  • 文献数据的云备份需求

我们想让一个软件能够跨平台、多设备的使用，那么这个软件应该要具备两个特性：

  • 数据文件独立
  • 数据文件对不同客户端状态的影响具有一致性

嘿嘿，这两点 Zotero 都能满足（其他的也没用过，只是云分析了一下😺）：

  • Zotero 作为一个开源的、提供 Linux 系统软件的项目，其数据文件的设计应该是考虑到跨平台需要的（Win 和 Linux 下文件组织方式一致），也就是说你把 Win 环境下的数据文件 COPY 到 Linux 环境下，OK，Zotero 也能够正确恢复，二者保证了 Zotero 系统状态一致性

我们对软件工作机制的简单分析可以对 Zotero 有更加深入的理解，那么，Zotero 是如何管理我的文献的？

你的文献存储在 Zotero 根目录下的 Storage 文件夹中，每个文献用一个含有神秘字符串的文件夹进行归档，文件夹中包括：

  • 你的 PDF 文献、-cache：文献的索引、-info：文件的信息
  • 你的文献记录在 Zotero 根目录下，使用 sqllite 数据库存储。

因此，实现跨平台和多设备同步访问只需要——将 usr 目录下的 Zotero 文件夹加入同步盘同步大礼包！然后在不同设备上进行本一开始提到的配置即可。

OK，至少我的需求解决了，如果大家关于文献管理或者科研有什么心得或者建议欢迎和我交流哦
