如何生成网络流

更新于: 2019-04-15 | 分类于 SDN

传统网络中的二层帧、三层报文、四层 TCP流，UDP 数据报在 SDN 中都统称为流（flow）。流是 SDN 网络中最基本的东西，没想到的是，在创造流的过程中我也花了很多时间。

在搭建实验环境，编写控制器，测试网络连通性和网络性能等任务时都没有使用到“流”，只有到仿真的时候你才会切实地需要流，真实环境的流。正是做到这一步的时候我遇到了麻烦，一个逻辑上的麻烦。

什么是流啊？怎么生成流啊？怎么生成真实环境的流啊？

这三个问题困扰了我好几天，我所参考的 SDN 文章文档中经过东拼西凑还是能给出系统设计的详细情况，但是对于系统是如何生成和使用流的，似乎所有人都是“看，那里有一条流，我们将其输入系统即可”，文章上是这么写的，但是对于我的实验来说就是“看，那里什么都没有”。

因此我必须查阅更多的资料来找到大家创造流的蛛丝马迹。那么，流从哪里来呢？资料说，流从 iperf 来，流从 ping 中来。好家伙，iperf 不是使用 tcp 流测试网络带宽的吗？ping 不是使用 ICMP 测试网络连通性的吗？怎么还能用来生成流了？

带着这样的疑问我又思考了一段时间，恍然大悟。

原来答案一直就在流的定义里：传统网路中的二层帧、三层报文、四层 TCP流，UDP 数据报在 SDN 中都统称为流。根据仿真业务的需要生成特定的数据在 SDN 网络中就是我们所需要的流，因此，凡是能生成网络数据的工具都能用来生成流，比如:
